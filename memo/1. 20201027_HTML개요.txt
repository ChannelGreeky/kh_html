

@ 인터넷이란?
-> 전 세계의 컴퓨터가 서로 연결되어 정보를 교환할 수 있는 하나의
거대한 컴퓨터 통신망
-> 인터넷에서는 다양한 서비스들을 제공함
	- Web Service, FTP Service, Mail Service 등등
-> 프로토콜을 이용하여 통신 규약을 약속하여 통신함
	- web == HTTP, Mail = SMTP, POP3, IMAP  등등

@ 웹(Web) 이란?
-> 네트워크에 연결된 시스템의 "정보"를 한 위치에서 "검색" 할 수 있는 서비스
-> 1989년 CERN(유럽 입자 물리 연구소)의 팀 버너스 리(Tim Berners-Lee)에 
의해 개발 됨
-> Hyper Text를 통해 정보를 교환하는 서비스
	- Hyper Text : 하나의 웹 문서에서 제공하는 연결 주소(링크)를
	통해 다른 문서로 이동 하는 것

@ Web Version
- Web 1.0 : 정적인 검색 서비스 제공 (초기 서비스 버전)
- Web 2.0 : 동적인 검색 서비스 제공 (현재 가장 많이 사용되는 기본 서비스 버전)
	+ 지능형 웹 서비스 (사용자의 패턴을 분석하여 개인별 맞춤 정보
	검색 서비스를 제공)
	- 사용자에게 필요한 지식만을 자동으로 검색하여 보여줌
	- facebook, youtube, instagram 등등..

@ Web의 역사
-> 웹은 www라고 부르며 World Wide Web의 약자
-> 월드 와이드 웹은 유럽의 입자물리학 연구소에서 얻어지는 엄청난 양의
연구 결과 및 자료의 효율적인 공유를 목적으로 1989년 팀버너스리의 제안으로
개발되었음
-> 1990년도에 웹이 등장하였으며 www의 등장으로 인해 인터넷 또한
급속도로 발전하게 되었음
-> 웹을 이용하여 데이터를 주고 받기 위해 사용한 언어는 
HTML(Hyper Text Markup Language)를 사용

@ HTML (Hyper Text Markup Language)
-> Web Data(Web Page)를 전송하기 위해 사용되는 Hyper Text를 구성하는
Markup Language (마크업 언어)
-> Hyper Text(하이퍼 텍스트)
	- Hyper Link를 통해 구성된 다른 정보로 쉽게 연결시켜주는 구조화된 
	문서
-> Markup Language (마크업 언어)
	- 태그 등을 이용하여 문서나 테이터의 구조를 정의하는 언어


@ HTML의 역사

1. 유럽 공동 원자핵 연구소 (CERN)의 팀 버너스 리
	- 팀 버너스 리의 제안으로 웹이 초기 개발되기 시작
	- 추후 팀 버너스 리는 Web의 활성화를 위해 W3C 재단을 창설
	- W3C는 웹 표준을 지정하는 기관 (HTML 표준을 제정하는 단체)
	- W3C는 HTML 표준을 제정하지만 표준을 강제 하지는 않았음
	(그로 인하여 웹 브라우저 전쟁이 일어나게 됨)


2. 제1차 웹 브라우저 전쟁
	- 최초의 GUI 모습을 갖춘 '모자이크' 브라우저가 탄생
	- 모자이크는 발전하면서 모자이크 커뮤니케이션에서 넷스케이프
	커뮤니케이션으로 이름을 바꿈
		(웹 브라우저 명칭도 모자이크에서 넷스케이프로 바꿈)
	- 1994년 넷스케이프는 정식 버전을 발표하고 배포를 시작
	- 이에 맞서 마이크로소프트가 인터넷 익스플로러를 발표
	- 두 회사는 자신들의 웹 브라우저를 사용자가 많이 사용하게 하려고
	W3C의 표준을 무시하고 엄청나게 많은 기능을 추가해 
	서로의 웹 브라우저가 최고 점유율이 되도록 만들려고 함
	- 인터넷 익스플로러가 아무리 발악해도 넷스케이프를 이기지 못하자
	마이크로 소프트사는 윈도우 OS에 강제적으로 인터넷 익스플로러를
	설치하여 윈도우를 배포함
	- 추가적으로 마이크로소프트는 애플과의 계약을 통해 매킨토시의
	기본 웹 브라우저도 인터넷 익스플로러로 설정
	- 결국 넷스케이프는 인터넷 익스플로러의 점유율을 따라잡기 
	힘들어지고, 1998년에 아메리카 온라인에 인수 됨


3. 플러그인
	- 웹 브라우저 전쟁으로 인해 웹은 엄청난 속도로 발전하기 시작
	- 하지만 정작 웹 표준을 지정하는 W3C는 빠르게 대응하지 못함
	- 이에 불만을 느낀 기업들은 플러그인을 만듦
	(플러그인은 웹 브라우저와 연동되는 특정 프로그램을 사용자 PC에
	추가로 설치해 웹 브라우저 기능을 확장하는 방법)
	- 1996년부터 현재 어도비 플래시, 액티브엑스 등은 웹을
	점점 풍부한 기능을 가진 공간으로 만들어 줌
	- 그로 인하여 웹 브라우저를 통하여 동영상 시청, 음악 감상 및
	금융 업무 등을 볼수 있게 됨

4. 웹 2.0 시대 
	- 2000년대부터 액티브엑스를 기반으로 웹 애플리케이션이 제작 됨
	- 또한 플래시를 기반으로 일반 사용자를 위한 애니메이션들이 만들어 짐
		-> 졸라맨, 우비소년, 마시마로 등이 나오기 시작할 떄
	- 플래시와 액티브엑스는 점점 발전하게 됨
		-> 플래시는 지속적으로 개발되어 웹 애플리케이션을
	개발할 수 있는 플렉스로 발전 됨
		
	- 인터넷은 점점 사용자끼리 뭉쳐 새로운 콘텐츠를 개발할 수 있는
	공간으로 발전하게 되고, 사용자끼리 정보 및 콘텐츠를 올려 창조할 
	수 있게 되는 웹 2.0 시대를 맞이 하게 됨
		-> youtube, 위키피디아 등

5. WHATWG의 등장
	- 전세계 최고 점유율을 가진 인터넷 익스플로러는 사실상 
	W3C의 표준 웹 브라우저가 됨
	- 그러다 보니 모든 웹 사이트에 액티브엑스 플러그인이 들어가게 되면서
	웹 사이트는 무거워 지기 시작함 (보안성도 떨어지게 됨)
	- 이를 막고자 2004년 W3C 회의에서 모질라 재단과 오페라
	소프트웨어는 새로운 HTML 표준 제안서를 제출
		(그러나 웹의 혁명을 위한 기존의 지향점에서 위배 된다고
	판단한 W3C는 제안을 거부 )
	- 그리하여 인터넷 익스플로러를 제외한 웹 브라우저 제공 기업 
	(애플, 모질라, 오페라 소프트웨어)은 독자적으로 새로운 
	웹 표준 기관을 설립 (WHATWG 의 탄생)
	- WHATWG는 새로운 웹 표준으로 Web Application 1.0을 표준으로 
	작성
	- 이와 비슷한 시기에 W3C는 마이크로 소프트와 XHTML 2.0 표준을
	작성하였으나 기존의 표준과 너무 동떨어져 자연스레 개발자들은
	쓰기 어려운 XHTML 2.0 표준을 안쓰게 됨 
	(2009년에 XHTML 2.0은 폐기)
	- 결국 W3C는 새로운 웹 표준으로 WHATWG의 Web Application 1.0
	표준을 선택 
	- 그리고 W3C는 Web Application 1.0 표준을 HTML5 표준으로 
	변경하고 WHATWG와 함께 HTML5 표준을 작성
	- HTML5표준은 2012년에 초안을 발표하고 2014년 10월에 
	정식 권고안을 발표


@ 제 2차 웹 브라우저 전쟁
	- 결국 마이크로소프트와 W3C와 함께한 XHTML 2.0표준이 붕괴되고
	인터넷 익스플로러의 기능 문제가 대두 되었음
	(다른 웹 브라우저는 모두 최신 표준을 지원하는데 인터넷 익스플로러는
	오히려 최신을 지원하지 못하는 역현상이 발생)

	- 다른 브라우저 업체들은 웹 브라우저 점유율을 뒤집을 수 있는 
	기회들이 생겨나게 됨 (오예!)

	- 이 시기를 놓치지 않으려고 모든 웹 브라우저가 빠른 속도로
	웹 브라우저들을 개발하고 주기적인 업데이트를 진행 
	(구글의 크롬은 10주 단위로 버전 업 진행)
	- 실제 웹 브라우저 기업들은 개발 이외에도 여러 마케팅까지 하게 됨
	(국내에는 인터넷 익스플로러 점유율이 높아서 그동안 하지 않았음)
	- 심한 경우에는 해당 브라우저가 아니면 서비스에 접근을
	못하는 경우도 만들어서 강제로 사용하게끔 만들기도 함
	- 그리고 현재에는 웹 플랫폼 업체들도 브라우저 개발을 	
	하게 되었음 (ex. 네이버 whale 브라우저)


@ HTML5의 특징
-> 구조적 설계 지원 (시맨틱 웹 지원)
	* 시맨틱 : 이름만 보고도 역할을 짐작할 수 있는 것
-> 그래픽 및 멀티미디어 기능 강화
-> CSS3 / JavaScript 지원
-> 다양한 HTML5 API 제공
-> 모바일 웹 지원 / 장치 접근 가능 (배터리 정보, 카메라, GPS 등등)

@ HTML 구성 요소

<p align='center'> HTML/CSS </p>

< : 태그의 시작
p : 태그명
align : 속성
'center' : 속성값
HTML/CSS : 내부 문자 
</p> : 태그의 종료

태그 (tag) : '<' 와 '>'로 묶인 명령어
	시작태그 (<태그>) 와 종료태그(</태그>)를 한쌍으로 이용
	(종료 태그가 없는 경우도 있음)

속성 : 요소의 시작태그에만 사용 / 명령어 구체화 역할
	여러개의 속을 사용할 수 있으며 속성의 구분은 공백

속성값 : 속성이 가지는 값, 값 입력시 " " 혹은 ' '를 이용

@ HTML 주의사항
-> 태그는 대소문자를 구분하지 않음 (단, 소문자 권장)
-> 파일의 확장자는 반드시 html, htm 으로 설정할 것
-> 문자의 공백은 한 개만 인식 
	(화면상 공백을 여러개 적용하기 위해서는 특수기호나 특정 태그를
	활용 해야 함) - &nbsp;

@ HTML 기본 구조
<!doctype html>
<meta charset = "UTF-8">
<head>

</head>
<body>

</body>
</html>

- <!doctype> : 문서의 형식을 알려주는 태그
- <meta> : 메타데이터 태그 (charset은 현재 인코딩 데이터 값을 표현)
- <head> : 문서의 각종 정보와 문서 자체에 대한 설명 값
		기본 셋팅값이 적용되는 태그 영역
- <body> : 화면에 출력해서 사용자에게 보여주는 정보가 들어 있는 태그 영역
	head에 들어가는 태그를 제외하고 모든 태그를 사용
-<!-- --> : HTML 주석
	해당 태그안에 작성된 내용은 브라우저가 해당 부분을 해석하지 않고 넘어간다.
